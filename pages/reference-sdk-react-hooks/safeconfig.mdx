# `SafeConfig`

The `SafeConfig` type is used by the `SafeProvider` and some individual functions to use a different configuration than the one from the `SafeProvider`.

This is the `SafeConfig` type definition:

```typescript
type SafeConfig<
  Provider extends SdkStarterKitConfig['provider'] = SdkStarterKitConfig['provider'],
  Signer extends SdkStarterKitConfig['signer'] = SdkStarterKitConfig['signer'],
  Chain extends ChainType = ChainType
> = SdkStarterKitConfig & {
  chain: Chain;
  provider: Provider;
  signer: Signer;
} & (Provider extends string
    ? { transport: HttpTransport }
    : { transport: CustomTransport })

type ExistingSafeConfig = {
  safeAddress?: string;
  safeOptions?: never;
}

type PredictedSafeConfig = {
  safeAddress?: never;
  safeOptions?: SafeConfig;
}

type SdkStarterKitRootConfig = {
  provider: SafeProvider['provider'];
  signer?: SafeProvider['signer'];
}

type SdkStarterKitConfig = SdkStarterKitRootConfig &
  (ExistingSafeConfig | PredictedSafeConfig)
```

The Safe React hooks package provides a [`createConfig`](./createconfig.mdx) function that can be used to create a new configuration.

## `SafeConfigWithSigner`

This type is a variation of the `SafeConfig` type where the `signer` property is mandatory. It is used by all hooks that perform sign operations, such as [`useSendTransaction`](./usesendtransaction.mdx), [`useConfirmTransaction`](./useconfirmtransaction.mdx), [`useUpdateThreshold`](./useupdatethreshold.mdx) and [`useUpdateOwners`](./useupdateowners.mdx).

This is the `SafeConfigWithSigner` type definition:

```typescript
type SafeConfigWithSigner = SafeConfig & { signer: string }
```
